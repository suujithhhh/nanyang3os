1. Introduction to Graphs

A graph G = (V, E) consists of a set of vertices (nodes) V and a set of edges E connecting pairs of vertices. Graphs are one of the most versatile data structures in computer science, used to model networks, maps, social connections, dependencies, and more.

Formal Definition: A graph G = (V, E) where V is a finite set of vertices and E ⊆ V × V is a set of edges. For an undirected graph, edges have no direction. For a directed graph (digraph), each edge (u, v) has a source u and destination v.

1.1 Graph Terminology

  • Adjacent vertices: Two vertices u and v are adjacent if edge (u, v) ∈ E
  • Degree: The number of edges incident to a vertex. In directed graphs, in-degree and out-degree are tracked separately.
  • Path: A sequence of vertices v₁, v₂, …, vₖ where each consecutive pair is connected by an edge.
  • Cycle: A path that starts and ends at the same vertex.
  • Connected graph: An undirected graph where every pair of vertices has a path between them.
  • Weighted graph: A graph where each edge carries a numerical weight (cost, distance, capacity).

1.2 Graph Representations

Two common representations are used depending on graph density:

Representation Space Edge lookup Best for
Adjacency Matrix O(V²) O(1) Dense graphs
Adjacency List O(V + E) O(degree) Sparse graphs

For most practical graph problems encountered in this course, the adjacency list representation is preferred due to its space efficiency when E << V².

2. Graph Traversal

Graph traversal algorithms systematically visit every vertex in a graph exactly once. The two fundamental traversal strategies are Breadth-First Search (BFS) and Depth-First Search (DFS).

2.1 Breadth-First Search (BFS)

BFS explores the graph level by level, visiting all neighbours of a vertex before moving deeper. It uses a queue (FIFO) as its core data structure.

Algorithm: BFS(G, source s)for each vertex u ∈ V − {s}: colour[u] = WHITE; dist[u] = ∞; parent[u] = NIL colour[s] = GREY; dist[s] = 0; parent[s] = NIL Q = empty queue ENQUEUE(Q, s) while Q is not empty: u = DEQUEUE(Q) for each v adjacent to u: if colour[v] == WHITE: colour[v] = GREY dist[v] = dist[u] + 1 parent[v] = u ENQUEUE(Q, v) colour[u] = BLACK

Time Complexity: O(V + E)   Space: O(V)

Key property: BFS always finds the shortest path (in terms of number of edges) from the source to any reachable vertex in an unweighted graph. The dist[] array records these shortest distances.

2.2 Depth-First Search (DFS)

DFS explores as far as possible along each branch before backtracking. It uses a stack (implicitly via recursion, or explicitly) and assigns discovery/finish timestamps to each vertex.

Algorithm: DFS(G)for each vertex u ∈ V: colour[u] = WHITE; parent[u] = NIL time = 0 for each vertex u ∈ V: if colour[u] == WHITE: DFS-VISIT(G, u) DFS-VISIT(G, u): colour[u] = GREY; time++ disc[u] = time for each v adjacent to u: if colour[v] == WHITE: parent[v] = u DFS-VISIT(G, v) colour[u] = BLACK; time++ finish[u] = time

Time Complexity: O(V + E)

DFS produces a DFS forest — a set of DFS trees. The discovery and finish timestamps reveal important structural properties of the graph including back edges (cycles) and topological order.

Edge Type Condition (disc/finish times) Implication
Tree edgev is WHITE when discoveredNormal DFS progression
Back edgev is GREY when discoveredCycle exists in directed graph
Forward edgedisc[u] < disc[v], v is BLACKDirected graphs only
Cross edgedisc[u] > disc[v], v is BLACKDirected graphs only

3. Shortest Path Algorithms

Shortest path problems ask: given a weighted graph G and a source vertex s, find the minimum-weight path from s to every other vertex (single-source shortest path, SSSP).

3.1 Dijkstra's Algorithm

Dijkstra's algorithm solves SSSP for graphs with non-negative edge weights. It greedily selects the unvisited vertex with the smallest known distance at each step.

Algorithm: Dijkstra(G, w, s)INITIALISE-SINGLE-SOURCE(G, s) S = ∅ Q = MIN-PRIORITY-QUEUE(V) // keyed by dist[] while Q is not empty: u = EXTRACT-MIN(Q) S = S ∪ {u} for each vertex v adjacent to u: RELAX(u, v, w) // if dist[v] > dist[u] + w(u,v): // dist[v] = dist[u] + w(u,v) // parent[v] = u // DECREASE-KEY(Q, v, dist[v])
Implementation Time Complexity
Array (linear scan)O(V²)
Binary min-heapO((V + E) log V)
Fibonacci heapO(V log V + E)
Important limitation: Dijkstra's algorithm is incorrect when edge weights are negative. For graphs with negative weights, use the Bellman-Ford algorithm instead.

3.2 Bellman-Ford Algorithm

Bellman-Ford handles negative edge weights and can detect negative-weight cycles. It relaxes all edges V−1 times.

Time Complexity: O(VE) — slower than Dijkstra but more general. Returns FALSE if a negative-weight cycle is reachable from the source.

4. Minimum Spanning Trees

A spanning tree of a connected undirected graph G is a subgraph that is a tree and includes all vertices of G. A Minimum Spanning Tree (MST) is the spanning tree with the minimum total edge weight.

MST Property (Cut Property): For any cut (S, V−S) of graph G, the minimum weight edge crossing the cut is always included in some MST of G.

4.1 Kruskal's Algorithm

Kruskal's algorithm builds the MST by greedily adding the cheapest edge that does not form a cycle. It uses a Union-Find (Disjoint Set Union) data structure to detect cycles efficiently.

Algorithm: Kruskal(G, w)A = ∅ for each vertex v ∈ V: MAKE-SET(v) sort edges E in non-decreasing order of weight w for each edge (u, v) in sorted order: if FIND-SET(u) ≠ FIND-SET(v): A = A ∪ {(u, v)} UNION(u, v) return A

Time Complexity: O(E log E) (dominated by sorting)

4.2 Prim's Algorithm

Prim's algorithm grows the MST one vertex at a time, always adding the cheapest edge connecting the current tree to a vertex outside it. It is similar in structure to Dijkstra's algorithm.

Time Complexity: O(E log V) with a binary min-heap.

5. Common Exam Mistakes & Pitfalls

  • Using Dijkstra with negative weights — always check for negative edges first; use Bellman-Ford instead.
  • BFS vs DFS for shortest path — BFS gives shortest path by edge count in unweighted graphs; DFS does not.
  • Forgetting to initialise dist[] = ∞ — a common bug in Dijkstra implementations.
  • Back edges indicate cycles — in DFS, a back edge means a cycle exists; critical for cycle detection.
  • MST is not unique — when edge weights are not all distinct, multiple MSTs may exist with equal total weight.
